# 물류창고 출고영역 통합 설계안

## 1. 문서 목적

본 문서는 물류창고 관리자 관점에서 매장 발주 승인건과 연계되는 출고 기능의 목표, 화면 구조, 상태 모델, 데이터 모델, 액션 범위를 정리한 설계 문서다.

1차 구현은 매장 출고를 우선 대상으로 하며, 창고간 이동/순환재고 판매 출고는 유형 확장용 인터페이스만 유지한다.

## 2. 범위와 기본 방향

### 2.1 업무 범위

- 출고 리스트 조회
- 출고 상세 조회
- 출고 확정 처리
- 배송중 처리
- 과거 완료 출고 이력 조회

### 2.2 이번 버전 구현 범위

- 출고 대상은 `매장 발주 승인건`만 사용한다.
- 출고 상태는 `출고 준비중 / 배송중 / 완료` 3단계로 단순화한다.
- 완료 상태는 매장 입고 확정(`RECEIVED`) 이후 반영한다.
- 창고 화면에서 매장 입고 확정 액션은 제공하지 않는다.

### 2.3 설계 원칙

- 출고 전용 `warehouseOutbound` 스토어를 신설한다.
- 단, 상태 source of truth 는 기존 `storeOrders.inboundStatus`를 우선한다.
- 충돌 시 출고 스토어가 단독으로 상태를 저장하지 않고 원본 상태를 그대로 따른다.

## 3. 메뉴 구조와 라우트

- 상위 메뉴: `입/출고 관리`
- 하위 메뉴: `출고 관리`

라우트:

- `/warehouse/outbound`
- `/warehouse/outbound/:id`

## 4. 화면 설계

### 4.1 출고 리스트

- 상태 탭: `전체 / 출고 준비중 / 배송중 / 완료`
- 검색: `출고번호, 발주번호, 매장명, 대표상품`
- 기간 필터: `요청일시`
- 목록 컬럼:
  - 출고번호
  - 발주번호
  - 유형
  - 매장
  - 대표상품
  - 총 SKU
  - 총 수량
  - 상태
  - 요청일시

### 4.2 출고 상세

- 기본 정보: 출고번호, 발주번호, 창고, 매장, 출고유형, 요청일시, 상태
- 품목 정보: 품목코드, 상품명, 옵션, 카테고리, 요청수량, 단가
- 진행 이력: 상태 전환 이력 및 처리자
- 액션:
  - `출고확정`
  - `배송중 처리`
- `완료` 상태는 읽기 전용

## 5. 상태 모델

### 5.1 내부 상태

- `READY_TO_SHIP` : 출고 준비중
- `IN_TRANSIT` : 배송중
- `COMPLETED` : 완료

### 5.2 연계 규칙

- `storeOrders.inboundStatus = READY_TO_SHIP` -> `READY_TO_SHIP`
- `storeOrders.inboundStatus = IN_TRANSIT | ARRIVED` -> `IN_TRANSIT`
- `storeOrders.inboundStatus = RECEIVED` 또는 `storeOrders.status = COMPLETED` -> `COMPLETED`

## 6. 데이터 모델

### 6.1 WarehouseOutbound

- `outboundId`
- `orderId`
- `warehouseId`
- `warehouseName`
- `storeId`
- `storeName`
- `outboundType` (`STORE_OUTBOUND | WH_TRANSFER | CIRCULAR_SALE`)
- `status`
- `requestedAt`
- `confirmedAt`
- `confirmedBy`
- `completedAt`
- `totalSkuCount`
- `totalRequestedQuantity`
- `items[]`
- `outboundStatusHistory[]`

## 7. 스토어 액션

- `getOutboundList(filters)`
- `getOutboundById(outboundId)`
- `confirmOutbound(outboundId, actor)`
- `startTransit(outboundId, actor)`
- `getCompletedHistory(filters)`

## 8. 테스트 기준

- `APPROVED + READY_TO_SHIP` 대상이 출고 준비중 목록에 노출되어야 한다.
- `startTransit()` 호출 시 배송중으로 전환되어야 한다.
- 완료 건은 읽기 전용으로 노출되어야 한다.
- `/warehouse/outbound/:id` 상세 진입이 가능해야 한다.
- 매장 입고 화면의 기존 동작이 유지되어야 한다.
