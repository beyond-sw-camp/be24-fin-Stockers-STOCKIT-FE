# 물류창고_출고영역_구현_내역

## 1. 구현 개요

물류창고 `출고 관리` 화면을 placeholder 상태에서 실제 동작 가능한 `출고 리스트 / 출고 상세` 구조로 확장했다.

핵심 목표:

- 매장 발주 승인건 기준 출고 목록 조회
- 출고 상세 조회 및 상태 이력 확인
- 출고확정/배송중 처리 액션 제공
- 매장 입고 확정 이후 완료 상태를 읽기 전용으로 반영

## 2. 메뉴 및 라우트 반영

- `/warehouse/outbound`
- `/warehouse/outbound/:id`

관련 파일:

- `src/router/index.js`
- `src/views/warehouse/WarehouseOutboundView.vue`
- `src/views/warehouse/WarehouseOutboundDetailView.vue`

## 3. 스토어 구현

### 3.1 출고 스토어 신설

파일:

- `src/stores/warehouseOutbound.js`

반영 내용:

- 출고 전용 Pinia 스토어 추가
- `storeOrders` 데이터를 기반으로 출고 엔트리 투영
- 출고 타입 필드 추가 (`STORE_OUTBOUND`, `WH_TRANSFER`, `CIRCULAR_SALE`)
- 1차에서는 `STORE_OUTBOUND`만 실제 동작

### 3.2 출고 상태 모델

- `READY_TO_SHIP` : 출고 준비중
- `IN_TRANSIT` : 배송중
- `COMPLETED` : 완료

매핑 규칙:

- `storeOrders.inboundStatus`를 기준으로 출고 상태를 계산
- `RECEIVED` 또는 발주 `COMPLETED`는 출고 `COMPLETED`로 처리

### 3.3 스토어 액션

- `getOutboundList(filters)`
- `getOutboundById(outboundId)`
- `confirmOutbound(outboundId, actor)`
- `startTransit(outboundId, actor)`
- `getCompletedHistory(filters)`

동기화 원칙:

- 출고 상태 변경은 `storeOrders` 액션(`markReadyToShip`, `markInTransit`)을 호출
- 출고 스토어가 상태를 단독 저장하지 않고 원본 상태를 우선

## 4. 출고 리스트 구현

파일:

- `src/views/warehouse/WarehouseOutboundView.vue`

반영 내용:

- 상태 탭: `전체 / 출고 준비중 / 배송중 / 완료`
- 검색: 출고번호, 발주번호, 매장명, 대표상품
- 기간 필터: 요청일시
- 목록 컬럼: 출고번호, 발주번호, 유형, 매장, 대표상품, 총 SKU, 총 수량, 상태, 요청일시
- 행 클릭 시 출고 상세로 이동

## 5. 출고 상세 구현

파일:

- `src/views/warehouse/WarehouseOutboundDetailView.vue`

반영 내용:

- 기본 정보 영역
- 품목 테이블
- 진행 이력
- 액션 버튼:
  - `출고확정`
  - `배송중 처리`
- 완료 상태는 읽기 전용

## 6. 매장 입고 연계

- 출고 완료 판단은 매장 입고 확정(`RECEIVED`) 이후 반영
- 창고 화면은 배송 진행까지만 수행하고 최종 입고 완료는 매장 화면에서 처리
